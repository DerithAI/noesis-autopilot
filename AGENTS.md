## Actual Architecture (vs README)

The README describes a modular package (`noesis/core.py`, `noesis/db/`, `noesis/memory/`, etc.) that **does not exist**. The real codebase is:

- **`noesis_autopilot.py`** (388 lines) — standalone autopilot, stdlib + sqlite3 only. This is the real entry point.
- **`noesis/autopilot.py`** — imports from a `noesis` package that is mostly empty (only this file exists). **Do not use** unless you are expanding the package.
- **`noesis_insights.py`** — reads `autopilot_memory.db`, produces `noesis_insights.md` + JSON.
- **`noesis_watcher.py`**, **`noesis_engine.py`**, **`noesis_dashboard.py`** — companion scripts, standalone.

## Critical Repo Quirks

### Gitignore vs Reality
`.gitignore` ignores `*.db` and runtime artifacts, but **these files exist in the working tree** and are needed:
- `autopilot_memory.db` — SQLite episodic memory (created by `noesis_autopilot.py`)
- `nexus_memory.db` — Used by `tests/test_system.py` (tests fail if missing)
- `*.log`, `*.json` insight files — Generated by `noesis_insights.py`

Do **not** delete these unless you know you are resetting state.

### The `noesis/` Package Is Nearly Empty
```
noesis/
├── autopilot.py      # Only real file
└── test_system.py    # Tests that need nexus_memory.db
```
Everything else the README mentions (`core.py`, `bridge.py`, `db/`, `memory/`, `vector/`, `lattice/`, `tools/`) **does not exist**. If you need those modules, you must create them.

### Ollama Dependency
`forge-bot/`, `venture-swarm.py`, and `noesis_autopilot.py`'s Ollama bridge **require a running Ollama server** at `127.0.0.1:11434`. The `OLLAMA_HOST` env var is auto-fixed from `0.0.0.0` to `127.0.0.1` in bot code.

### Tests
```bash
# forge-bot tests (19 passing, includes bridge + MCP memory imports)
cd forge-bot && python -m pytest test_bot.py -v

# noesis tests (require nexus_memory.db with episodes table)
cd tests && python -m pytest test_system.py -v
# ^ Will fail if nexus_memory.db missing or empty
```

### Generated Artifacts
`forge-*/` directories are gitignored but are created by `venture-swarm.py` and `forge.py`. The **only** forge artifact kept in git is `forge-bot/` (explicit `!forge-bot/` exception in `.gitignore`). `ventures/` deploy packages are also kept.
