# EVO-HUB Design System

> Generated by integrating Open Design craft rules with EVO-FORGE identity.

## Design Tokens

### Colors
```css
:root {
  --evo-primary: #0ea5e9;       /* Sky 500 - cognitive accent */
  --evo-secondary: #8b5cf6;   /* Violet 500 - AI purple */
  --evo-accent: #f59e0b;      /* Amber 500 - warning/energy */
  --evo-surface: #0f172a;     /* Slate 900 - dark background */
  --evo-text: #f8fafc;        /* Slate 50 - primary text */
  --evo-muted: #94a3b8;       /* Slate 400 - secondary text */
  --evo-border: #334155;      /* Slate 700 - borders */
  --evo-success: #22c55e;     /* Green 500 - pass */
  --evo-danger: #ef4444;      /* Red 500 - fail */
}
```

### Typography
- **Font**: Inter (sans-serif) for UI, JetBrains Mono for code
- **Scale**: 12/14/16/18/24/32/48
- **Line-height**: 1.5 for body, 1.2 for headings

### Spacing
- **Base**: 4px grid
- **Scale**: 4, 8, 12, 16, 24, 32, 48, 64

## Craft Rules (from Open Design)

### Accessibility
- Contrast ratio ≥ 4.5:1 for normal text
- Focus rings on all interactive elements
- Semantic headings (h1→h6, no skips)
- `prefers-reduced-motion` support

### Animation
- Micro-interactions: 150–300ms
- Easing: `cubic-bezier(0.23, 1, 0.32, 1)` for enter
- Exit faster than enter (~60% duration)
- No decorative-only animation

### State Coverage
Every interactive element has:
- Default, hover, active, disabled states
- Loading state with skeleton/shimmer
- Error state with recovery path
- Success state with confirmation

## Component Patterns

### Cards
- Background: `--evo-surface` with 1px `--evo-border`
- Border-radius: 8px
- Shadow: `0 1px 3px rgba(0,0,0,0.3)`
- Padding: 16px

### Buttons
- Primary: `var(--evo-primary)` background, white text
- Secondary: transparent, `var(--evo-primary)` border
- Danger: `var(--evo-danger)`
- Touch target: ≥44×44px

### Charts
- Use accessible colors (not red/green alone)
- Tooltips on hover/tap
- Legend always visible
- Direct labeling for small datasets

## EVO-Specific Patterns

### Cognitive Loop Visualization
```
[Perception] → [Intention] → [Context] → [Reasoning] → [Response] → [Direction]
```
- Each stage: node with status indicator
- Active stage: pulse animation
- Completed: checkmark
- Failed: error icon

### ITDD Tracker
```
[RED] → [GREEN] → [REFACTOR]
```
- Red: failing test count
- Green: passing test count
- Refactor: code quality score

### Memory Bridge
- SQLite (NOESIS) ↔ ChromaDB (LUMEN)
- Migration progress bar
- Hybrid search toggle
